MetaSearch Pro Documentation

Welcome to the MetaSearch Pro documentation. This guide will help you get started with the application and learn how to create custom connectors.

Installation

Prerequisites

  • Windows 10 or Windows 11
  • .NET 10 Runtime

Build from Source

git clone <repository-url>
cd MSCC
dotnet build
dotnet run

Quick Start

  1. Launch MetaSearch Pro
  2. Configure your first data source using the + button
  3. Select the data sources you want to search
  4. Enter a search term and press Enter
  5. View results and add labels as needed

Configuration

MetaSearch Pro stores its configuration in the following locations:

File Location Description
settings.json %APPDATA%\MSCC\ Application settings (language, UI preferences)
datasources.json %APPDATA%\MSCC\ Data sources and groups configuration
Scripts\ %APPDATA%\MSCC\ Custom script-based connectors

Connectors

Connectors are plugins that enable MetaSearch Pro to search specific types of data sources. Each connector defines:

  • Configuration parameters required to connect
  • Search logic to find and return results
  • Detail view configuration for displaying results
  • Optional actions that can be performed on results

Data Sources

A data source is an instance of a connector with specific configuration. For example, you can create multiple file system data sources:

  • Documents - searching your Documents folder
  • Downloads - searching your Downloads folder
  • Project Files - searching a specific project directory

Groups

Groups help you organize data sources into logical collections. You can select an entire group to search all its data sources at once.

Labels & Saved Queries

Labels allow you to tag search results with custom keywords. This creates a personal taxonomy across all your data sources.

Saved queries store your search term along with selected data sources and any labels you've applied, allowing you to quickly repeat complex searches.

Script Plugins

Script plugins are C# files that are compiled at runtime. They're ideal for quick prototypes and simple connectors.

Creating a Script Plugin

  1. Open Plugins → Script Manager
  2. Click + New Script
  3. Edit the generated template
  4. Click Compile to register the connector

Script Template

public class MyConnector : ScriptedConnectorBase
{
    public override string Id => "my-connector";
    public override string Name => "My Connector";
    public override string Description => "My custom connector";
    public override string Version => "1.0.0";

    public override IEnumerable<ConnectorParameter> ConfigurationParameters => new[]
    {
        new ConnectorParameter
        {
            Name = "ApiUrl",
            DisplayName = "API URL",
            IsRequired = true
        }
    };

    public override async Task<IEnumerable<SearchResult>> SearchAsync(
        string searchTerm,
        int maxResults = 100,
        CancellationToken cancellationToken = default)
    {
        var results = new List<SearchResult>();
        
        // Your search logic here
        
        return results;
    }
}

Compiled Plugins

For complex connectors with dependencies, you can create compiled plugins using Visual Studio.

Steps

  1. Create a new Class Library project targeting .NET 10
  2. Add a reference to the MSCC project
  3. Implement the IDataSourceConnector interface
  4. Register your connector in DataSourceManager.cs
